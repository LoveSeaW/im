# 编译与部署流程

## 环境要求

- **Go** 1.21+（后端）
- **Node.js** 16+ / npm（前端）
- **Docker** + Docker Compose

---

## 一、后端编译

### 1.1 整体架构

后端基于 Go + go-zero 框架，所有服务通过一个统一的 `Dockerfile` 编译为二进制，再按服务拆分为独立镜像。

```
backend/
├── Dockerfile              # 基础编译镜像（编译所有 Go 二进制）
├── deploy/
│   ├── docker-compose.yaml # 编排所有服务
│   ├── fim_chat/chat_api/  # 聊天 API 服务
│   ├── fim_chat/chat_rpc/  # 聊天 RPC 服务
│   ├── fim_group/          # 群聊相关
│   ├── fim_user/           # 用户相关
│   ├── fim_auth/           # 认证服务
│   ├── fim_file/           # 文件服务
│   ├── fim_logs/           # 日志服务
│   ├── fim_settings/       # 设置服务
│   ├── fim_gateway/        # 网关
│   ├── fim_web/dist/       # 前端静态文件
│   ├── nginx/              # Nginx 配置
│   └── postgres/           # 数据库初始化
```

### 1.2 服务列表

| 服务 | 容器名 | 说明 |
|------|--------|------|
| gateway | im_gateway | API 网关 |
| auth_api | im_auth_api | 认证服务 |
| chat_api | im_chat_api | 聊天 API |
| chat_rpc | im_chat_rpc | 聊天 RPC |
| group_api | im_group_api | 群聊 API |
| group_rpc | im_group_rpc | 群聊 RPC |
| user_api | im_user_api | 用户 API |
| user_rpc | im_user_rpc | 用户 RPC |
| file_api | im_file_api | 文件 API |
| file_rpc | im_file_rpc | 文件 RPC |
| logs_api | im_logs_api | 日志 API |
| settings_api | im_settings_api | 设置 API |
| settings_rpc | im_settings_rpc | 设置 RPC |

基础设施：PostgreSQL、Redis、etcd、ZooKeeper、Kafka、Nginx

### 1.3 编译步骤

```bash
# 进入后端目录
cd backend

# 步骤一：编译基础镜像（编译所有 Go 二进制，约 1-2 分钟）
docker build --no-cache -t fim_server -f Dockerfile .

# 步骤二：进入部署目录，构建所有服务镜像
cd deploy
docker compose build --no-cache

# 步骤三：启动所有容器
docker compose up -d
```

**如果只修改了部分服务代码，可仅重建受影响的服务：**

```bash
# 重编译基础镜像（始终需要，因为 Go 是静态编译）
docker build --no-cache -t fim_server -f backend/Dockerfile backend/

# 仅重建指定服务
cd backend/deploy
docker compose build --no-cache chat_api group_api
docker compose up -d chat_api group_api
```

### 1.4 常用运维命令

```bash
cd backend/deploy

# 查看所有容器状态
docker compose ps

# 查看某服务日志
docker logs -f im_chat_api
docker logs -f im_group_api
docker logs --tail=50 im_chat_api

# 重启某服务
docker compose restart chat_api

# 停止所有服务
docker compose down

# 完全重建（含基础设施）
docker compose down
docker compose up -d
```

## 二、前端编译

### 2.1 技术栈

Vue 3 + TypeScript + Vite + Element Plus

### 2.2 编译步骤

```bash
# 进入前端目录
cd frontend

# 安装依赖（首次）
npm install

# 编译生产版本
npm run build

# 产出目录：frontend/dist/
```

### 2.3 部署前端

前端编译产物需复制到后端的部署目录，Nginx 容器会挂载该目录：

```bash
# 编译后复制到部署目录
cp -r frontend/dist/* backend/deploy/fim_web/dist/

# Nginx 容器通过 volume 挂载，新文件即时生效，无需重启
# 配置位于：backend/deploy/nginx/nginx.conf
```

---

## 三、完整一键构建

```bash
#!/bin/bash
set -e

# 1. 前端
cd frontend
npm run build
cd ..

# 2. 复制前端产物
cp -r frontend/dist/* backend/deploy/fim_web/dist/

# 3. 后端基础镜像
docker build --no-cache -t fim_server -f backend/Dockerfile backend/

# 4. 构建所有服务镜像
cd backend/deploy
docker compose build --no-cache

# 5. 启动
docker compose up -d

echo "部署完成"
```

## 四、目录权限说明

构建前确保以下目录不被 `.dockerignore` 排除且有正确权限：

```
# backend/.dockerignore
deploy/postgres/datadir      # 排除 PostgreSQL 数据目录
deploy/fim_web/dist          # 排除前端（由宿主机挂载）
deploy/nginx/logs            # 排除日志
deploy/fim_file/file_api/uploads  # 排除上传文件
```
